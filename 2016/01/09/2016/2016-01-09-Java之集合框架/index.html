Java之集合框架

Posted on 2016-01-09 Edited on 2022-11-29 In code
集合框架该有的都有了

在Java中集合是非常重要的一大块，文章从Collection展开，再涉及到泛型和Map集合，大致结构如下：概述、Collection、泛型、Map集合、集合框架工具类。

概述

集合类：是存储对象最常用的一种方式

集合和数组的区别

  • 数组：可以存储基本数据类型和对象，但长度是固定的
  • 集合：只能存储对象（可以是不同类型的对象），长度是可变的

Collection

集合框架根接口

共性方法

新增：集合中存储的都是对象的引用(即在集合中新增一份地址)

  • add(E e)
  • addAll(Collection<? extends E> c)

删除

  • remove(Object o)
  • removeAll(Collection<?> c)
  • clear() :清空集合

判断

  • contains(Object o)
  • containsAll(Collection<?> c)
  • isEmpty()

取交集

  • retainAll(Collection<?> c)

迭代器：Iterator

  • hasNext()
  • next()
  • remove()

List集合

特有方法

凡是可以操作角标的方法都是该体系的特有方法

增加

  • add(int index, E element)
  • addAll(int index, Collection<? extends E> c)

删除

  • remove(int index)

修改

  • set(int index, E element)

查询

  • get(int index)
  • listIterator() / listIterator(int index) -> List特有的迭代器，在迭代的过程中，可CRUD
  • subList(int fromIndex, int toIndex)

ArrayList

底层使用数组数据结构。默认长度10，按1.5倍增长

特点：查询速度快，增删速度慢，线程不同步

练习

1、去除ArrayList中的重复元素

2、将自定义对象作为元素存入到ArrayList集合中，并去除重复元素。重要结论：List集合判断元素是否相同，依据是元素的equals方法

LinkedList

底层使用链表数据结构。

特点：增删速度快，查询速度慢

特有方法

  • addFirst(E e)
  • addLast(E e)
  • getFirst()
  • getLast()
  • removeFirst()
  • removeLast()

但是如果集合中没有元素，会抛出NoSuchElementException，从JDK1.6开始出现了替代方法。集合中没有元素，则返回null。

  • offerFirst(E e)
  • offerLast(E e)
  • peekFirst()
  • peekLast()
  • pollFirst()
  • pollLast()

练习：使用LinkedList模拟堆栈或队列数据结构

Vector类

底层使用数组数据结构。除了线程同步外，其他内容同ArrayList。

特点：查询速度快，增删速度慢，线程同步。

Set集合

元素是无序的，元素不可能重复。”无序”指存入和取出的顺序不一定一致。

HashSet

底层数据结构是哈希表,线程不同步

HashSet是如何保证元素唯一性的？

  • 是通过元素的两个方法：hashCode和equals
  • 如果元素的HashCode值相同，才回判断equals是否为true
  • 如果元素的HashCode值不同，不会调用equals

练习：将自定义对象作为元素存入到HashSet集合中，并去除重复元素

TreeSet

底层数据结构是二叉树

特点：默认对集合中元素排序

TreeSet是如何保证元素唯一性的？

  • compare方法 return 0 表示是重复元素
  • compareTo方法 return 0 表示是重复元素

练习：往TreeSet集合中存储自定义对象学生，按照学生的年龄进行排序。在排序时，当主要条件相同时，一定要判断次要条件。

排序方式

  • 第一种：让元素自身具备比较性。要求元素实现Comparable接口，并覆盖compareTo方法，这种方式也成为自然顺序或默认顺序

  • 第二种：让容器自身具备比较性。在容器初始化时(构造函数中)就让其具有比较方式，具体做法：定义一个类(作为比较器)，实现Comparator接口，并覆盖compare方法。

当两种排序都存在时，以”比较器”为主，推荐采用”比较器”的方式开发编程。

Map集合

共性方法

添加

  • put(K key, V value)，返回前一个与key关联的值或者null
  • putAll(Map<? extends K,? extends V> m)

删除

  • remove(Object key)
  • clear()

判断

  • containsKey(Object key)
  • containsValue(Object value)

获取

  • get(Object key)
  • size()
  • values()
  • entrySet() -> 将Map集合中的映射关系取出存入到Map.Entry中
  • keySet() -> 将map中所有的键存入到Set集合中，再迭代取出所有键，然后利用get()方法获取每一个值

Hashtable类

底层数据结构哈希表，不可以存入null键null值，线程同步，jdk1.0，效率低

HashMap类

底层数据结构哈希表，可以存入null键null值，线程不同步，jdk1.2，效率高

TreeMap类

底层数据结构是二叉树，线程不同步，可用于给Map集合中的键排序

练习：获取字符串中字母出现的次数

集合框架工具类

Collections

排序

  • sort(List list)

  • sort(List list, Comparator<? super T> c)

获取最大值

  • max(Collection<? extends T> coll)

  • max(Collection<? extends T> coll, Comparator<? super T> comp)

二分查找

  • binarySearch(List<? extends Comparable<? super T>> list, T key)

  • binarySearch(List<? extends T> list, T key, Comparator<? super T> c)

替换反转

  • fill(List<? super T> list, T obj)

  • replaceAll(List list, T oldVal, T newVal)

  • reverse(List<?> list)

  • reverseOrder()

  • reverseOrder(Comparator cmp)

  • swap(List<?> list, int i, int j)

线程同步

  • synchronizedList(List list)

Arrays

用于操作数组的工具类

  • asList(T… a) 将数组变成集合后，不可以使用集合的增删方法
  • toString(byte[] a)
